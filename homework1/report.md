## 二、程序设计与实现思路

### 1. 项目架构
项目架构如下：
```
homework1/
├── src/              # 源代码目录
│   ├── main.py       # 主程序入口，负责菜单交互
│   ├── student_manager.py  # 核心业务逻辑（增删改查）
│   ├── utils.py      # 工具函数（格式化输出等）
│   └── constants.py  # 常量定义（文件路径、分隔符等）
├── data/             # 数据文件目录
│   └── students.txt  # 学生记录存储文件
└── README.md         # 运行说明
```

### 2. 核心流程
1. 程序启动 → 加载 students.txt 数据
2. 显示菜单 → 用户选择操作
3. 根据选择调用对应功能：
- 添加：检查学号唯一性 → 写入文件
- 查询：按条件检索 → 显示结果
- 修改：查找记录 → 修改 → 重写文件
- 删除：查找记录 → 临时文件过滤 → 覆盖原文件
- 显示：读取所有记录 → 格式化输出
4. 循环菜单，直到用户退出

### 3. 功能实现细节

1. 新增学生记录
**功能要求**：
- 提示用户输入：学号、姓名、专业、年级
- 检查学号是否已存在（去重校验）
- 将记录追加写入 students.txt 文件
**边际/错误情况**：
- 学号为空：提示“学号不能为空”
- 学号格式错误（如非数字、长度不符合要求）：提示“学号格式错误，请输入正确的学号”
- 学号已存在：提示“学号已存在，不允许重复添加”
- 姓名为空：提示“姓名不能为空”
- 专业为空：提示“专业不能为空”
- 年级格式错误（如非数字、非合理年份）：提示“年级格式错误，请输入正确的年份”

2. 查询学生记录
**功能要求**：
- 按学号查询：输入学号，显示该学生的详细信息
- 按姓名查询：输入姓名，显示所有同名学生的信息（模糊或精确匹配均可）
**边际/错误情况**：
- 查询学号为空：提示“请输入学号”
- 学号不存在：提示“未找到该学号的学生记录”
- 查询姓名为空：提示“请输入姓名”
- 姓名不存在：提示“未找到该姓名的学生记录”
- 文件读取失败（如文件不存在）：提示“文件读取失败，请检查文件是否存在”

3. 修改学生记录
**功能要求**：
- 输入待修改学生的学号，若学生存在，允许修改其专业或年级
- 修改原理：读取文件 → 修改对应记录 → 重写文件
**边际/错误情况**：
- 查询学号为空：提示“请输入学号”
- 学号不存在：提示“未找到该学号的学生记录”
- 修改后的专业为空：提示“专业不能为空”
- 修改后的年级格式错误：提示“年级格式错误，请输入正确的年份”
- 文件读写失败：提示“文件操作失败，请检查权限或文件状态”

4. 删除学生记录
**功能要求**：
- 输入待删除学生的学号
- 找到该记录并删除
- 删除原理：创建临时文件 → 拷贝除待删除记录外的数据 → 用临时文件覆盖原文件
**边际/错误情况**：
- 查询学号为空：提示“请输入学号”
- 学号不存在：提示“未找到该学号的学生记录”
- 文件操作失败（如文件权限不足、临时文件创建失败）：提示“文件操作失败，请检查权限或文件状态”
