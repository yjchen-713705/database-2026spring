# 上机作业1
## 作业：基于文件系统的简单学籍管理程序
### 1. 作业目标
1. 理解数据持久化：掌握如何将数据永久存储到磁盘文件中。
2. 体验文件系统的局限性：通过手动实现数据的增删改查，体会直接使用文件系统管理数据时面临的数据冗余、数据不一致性、以及应用程序对数据结构的强依赖性等问题。
3. 编程实践：提高使用C语言(文件I/O)或Python语言(文件操作)解决实际问题的能力，为后续学习SQL和数据库设计打下基础。

### 2. 作业要求
请使用C语言或 Python语言完成一个简单的学籍管理命令行程序。程序必须直接操作文本文件(如.txt或.csv)来存储数据，禁止使用任何数据库系统(如MySQL、SQLite、Redis等)或数据库驱动库。

#### 基本功能要求
程序启动后，应显示一个交互式菜单，支持以下功能：
1. 添加学生记录
- 提示用户输入：学号、姓名、专业、年级。
- 将记录追加写入到指定的数据文件(例如students.txt)中。
- 注意事项：需要检查学号是否已存在(简单的去重校验)。
2. 查询学生记录
- 按学号查询：输入学号，显示该学生的详细信息。
- 按姓名查询：输入姓名，显示所有同名学生的信息(模糊或精确匹配均可)。
3. 修改学生信息
- 输入待修改学生的学号。
- 如果该学生存在，允许用户修改其专业或年级。
- 修改原理：通常需要将文件全部读入内存，修改对应记录后，将所有数据重新写回文件(或者是修改特定字节，但初学者建议采用"读取-修改-重写"的方式)。
4. 删除学生记录
- 输入待删除学生的学号。
- 找到该记录并删除。
- 删除原理：通常创建一个临时文件，将除了待删除记录以外的所有数据拷贝到临时文件，然后用临时文件覆盖原文件。
5. 显示所有记录
- 以表格形式清晰打印当前文件中的所有学生记录。

#### 数据存储格式要求
数据文件应为纯文本文件，命名为students.txt。
每行存储一条学生记录，字段之间用特定的分隔符(例如逗号，或竖|)分隔。
示例(CSV格式)：
```
2024001,张三,计算机科学,2024
2024002,李四,软件工程,2024
2024003,王五,数据科学,2023
```

### 3. 提交内容&时间要求

2. 可执行程序/运行说明：如果是C语言，建议提交编译后的exe文件；如果是Python，请确保助教有运行环境，并提交源代码即可。
3. 数据文件：students.txt(应包含至少3条初始测试数据)。
4. 实验报告(PDF格式)：
- 简述程序设计与实现思路。
- 列出程序运行截图。
- 思考题(重点)：
  - 当你修改了程序中的结构体定义(例如增加一个"班级"字段)后，原有的students.txt文件还能直接使用吗？如果不修改程序，会有什么后果？这说明了文件系统的什么问题？
  - 如果两个用户同时操作这个程序(例如一个在查询，一个在修改)，可能会发生什么问题？
